Feature Engineering Code

# Rolling statistics (window = 10 cycles)
for sensor in sensor_cols:
    df[f'{sensor}_rmean'] = grp[sensor].transform(
        lambda x: x.rolling(10, min_periods=1).mean())
    df[f'{sensor}_rstd']  = grp[sensor].transform(
        lambda x: x.rolling(10, min_periods=1).std().fillna(0))

# Normalized cycle position (0=start, 1=failure)
df['cycle_norm'] = df['cycle'] / grp['cycle'].transform('max')

# Health index: aggregate degradation signal
df['health_index'] = 1 - df[rmean_cols].mean(axis=1)
Total features after engineering: 36 (21 raw + 10 rolling + 3 op conditions + 2 derived)
STEP 06

Machine Learning Models

Linear Regression

Baseline model
Ordinary least squares. Fast, interpretable. Assumes linear relationship between features and RUL.

Ridge Regression

Regularized baseline
L2 regularization (α=10) prevents overfitting on correlated sensor features.

Random Forest

★ Best performer
200 trees, max depth 10. Handles nonlinear degradation patterns. Provides feature importances.

Gradient Boosting

XGBoost-equivalent
300 estimators, learning rate 0.05. Sequential error correction. High accuracy, slower training.
Validation Strategy: Time-aware train/test split (no shuffling) to prevent data leakage. Test set = last 20% of unseen engines. RUL predictions clipped to [0, 125] range.

Urgency Classification Logic

def classify_urgency(predicted_rul, safety_margin=15):
    if   rul <= safety_margin:          return 'CRITICAL'  # immediate action
    elif rul <= safety_margin * 3:      return 'HIGH'      # schedule soon
    elif rul <= safety_margin * 6:      return 'MEDIUM'    # plan ahead
    else:                              return 'LOW'       # monitor only

# Minimize: planned_maintenance_cost + emergency_repair_cost

STEP 09

Conclusions & Future Work

✅ Key Findings

  • Random Forest achieves R² = 0.9398, RMSE = 9.95 cycles
  • Rolling mean features are the most predictive — more than raw sensors
  • Health index (aggregate degradation) is consistently in top 5 features
  • AI-driven maintenance yields ~5× cost reduction vs reactive maintenance
  • All 4 models outperform random baseline — R² > 0.91 across the board

🔮 Future Work

  • LSTM / Transformer models for sequential time-series patterns
  • Uncertainty quantification (Bayesian / conformal prediction)
  • Multi-objective optimization with OR-Tools / Pyomo
  • Real-time streaming inference (Apache Kafka + model serving)
  • Anomaly detection as a complementary signal to RUL
  • Transfer learning across different engine types (FD002, FD003, FD004)

Project Summary

This project demonstrates a complete end-to-end predictive maintenance pipeline: from raw turbofan sensor data through preprocessing, feature engineering, and comparative ML model training, to actionable maintenance scheduling. The best model (Random Forest) predicts remaining useful life within ~10 cycles RMSE across 4,987 test observations from 20 unseen engines. The maintenance optimizer translates these predictions into a priority-ranked schedule that reduces emergency failures and minimizes total operational cost — demonstrating applied AI value in industrial settings.
